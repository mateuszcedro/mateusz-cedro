Classification Project

Goal

The goal of this project was to analyze the characteristics of brain stroke patients dataset and compare performance of various tree-based models, in order to pick the best-performing one.

Dataset

The dataset contains medical information about 4981 patients, 248 of which suffered from brain stroke. Patients informations include basic personal information like gender and age, work type, residence type or indication if patient was ever married and basic medical information like BMI index, average glucose level, smoking status, heart diseases and hypertension history. The dataset comes from Kaggle and was published with Public Domain License (https://www.kaggle.com/datasets/jillanisofttech/brain-stroke-dataset)

Models

Classification Trees (CT)

Random Forest (RF)

Extreme Gradient Boosting (XGB)

We have tried with Adaboost too, as it is suited for binary classification problems, although limitations of our personal machines

Approach

After initial analysis we have transformed average glucose level into logarithmic version due to skewed distribution. The biggest issue we were facing was imbalanced dataset, containing only 4.9789199 % of success values (brain strokes). We have tried using SMOTE algorithm in order to upsample the minority group and compared the results with applying weights to models. In XGBoost model we have used SMOTE as a part of cross validation. We have tried both common values of hyperparameters and best practices knowledge, followed by hyperparameter tuning. At the end we have compared performed three types of models.

Evaluation Metric

Evaluation of calculated models was performed with Gini Score. In situations, where the Gini Score was similar for two models, we have also considered comparison of Gini Score for train and test data (in order to avoid overfitting) and the ROC Curve, as the treshold in this case should rather be set to maximise Sensitivity.

Conclusion and Final remarks

Throughout this project, we have overcame the issue of imbalanced dataset with SMOTE algorithm and weights of observations, identified best model for each category and concluded, that for predicting brain stroke of patient, the best-suited model from considered, was the most advanced one, so Extreme Gradient Boosting. It is worth to acknowledge, that AdaBoost method could also be beneficial to consider as a method designed for binary classification.

Classification Trees

Classification trees models included default hyperparameters model tree1, which became overfitted, tree2 model with more common parameters and assigned weights, tree3 model with beforehand generated, upsampled train dataset obtained with SMOTE algorithm and tree4p model with overgrown and pruned weights-based tree.

It seems like Tree2 obtained the best results obtaining Gini coefficient at the level of 0.598 at test sample. Although it’s worth to mention, that pruned tree has the smaller difference between train and test sample, so there is smaller risk of overfitting for future purposes. Also in this case, we are rather interested in high sensitivity than specificity, and for specific tresholds, specificity is higher for tree4p. There is also a possibility of combining the trees, but for now we will stick wiht tree4p as it gets the most consistent results.

Random forest

For the Random Forest method, we started with building the default RF model, which turned out to be overfitted. Basing on our experience and known best practices, we’ve assigned hyperparameters and built two additional models. One based on sampling weights and another on SMOTE-upsampled data. ROC curve suggests that the models behave similarly with slight advantage of RF2 (weights method) in terms of Gini Score. Although there is significantly smaller difference between train and test sample for RF3 (SMOTE), so we will continue analysis using this solution.

Extreme Gradient Boosting method

As the last method, we have checked the XGB method. This time we started already with upsampling with SMOTE algorithm inside the cross validation. At first we have set up the hyperparameters according to our best practice knowledge and then we did the hyperparameter tuning. Due to limitations of our machines, we performed the tuning parameter by parameter in following order: 1. Number of iterations, 2. Maximum tree depth, 3. Minimal child weight, 4. Predictors used for splits (colsample_bytree), 5. Size of subsample, 6. Learning rate coefficient. Based on the train sample Gini, XGB7 model, so the last iteration, performed best, although comparing Gini score for Test sample, it turned out, that model 4, so the one with optimal child weight was the optimal one and following optimizations weren’t improving performance. On the ROC Curve chart, we can see that the models performance vary for different treshold levels, although it seems like XGB4 model is performing best. Especially as its the best performing model for higher sensitivity levels, which we are interested in.

Final Comparison

For Final comparison, we have picked the best performing models of each type. As could be expected, the best performance was achieved by the most complicated model, which was Extreme Gradient Boosting. In further examples, it could be beneficial to check performance of AdaBoost model and Neural Networks, but this time we were quite limited by computational power.
